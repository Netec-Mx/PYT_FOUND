# **Python Moderno y Automatización**



## Plataforma de laboratorios



Te damos la bienvenida a la **plataforma de laboratorios** del curso **Python Moderno y Automatización**. Aquí podrás explorar diferentes tecnologías a través de prácticas guiadas. ¡Desarrolla tus habilidades y lleva tus conocimientos al siguiente nivel!



## Lista de laboratorios



Cada uno de estos laboratorios está diseñado para ofrecerte una experiencia práctica. Haz clic en los enlaces para comenzar.


### [Laboratorio 1. Análisis de Logs JSON de un Servicio en la Nube](./Capítulo 1/README.md)

  - **Descripción**: Al finalizar la práctica, serás capaz de manejar estructuras de datos avanzadas en Python, como listas, diccionarios y sets, para procesar eficientemente registros en formato JSON. Asimismo, podrás aplicar técnicas de filtrado, transformación y agregación de datos que te permitirán extraer información relevante de manera estructurada. Finalmente, desarrollarás el criterio necesario para seleccionar la estructura de datos más adecuada según la tarea, dejando preparada la información para su integración y manipulación en APIs REST en prácticas futuras..

  - ⏱️ **Duración estimada**: 20 min.



### [Laboratorio 2.Cliente de Métricas de Nube con Validación](Capítulo 2/README.md)

  - **Descripción**: Al finalizar la práctica, serás capaz de diseñar un conjunto de funciones en Python organizadas como un “mini-SDK”, orientado a estructurar de manera clara y reutilizable los procesos de datos. Además, utilizarás parámetros flexibles (*args, **kwargs), tipado estático (Optional, List, Dict) y retornos múltiples para construir funciones con contratos bien definidos de entrada y salida. Como resultado, podrás implementar un flujo completo que permita cargar métricas desde un archivo JSON, validarlas, procesarlas estadísticamente y simular su envío a un endpoint REST, integrando así buenas prácticas de desarrollo y preparación para escenarios reales..

  - ⏱️ **Duración estimada**: 20 min.



### [Laboratorio 3. SDK de Configuración y Métricas Robusto](Capítulo 3/README.md)

  - **Descripción**: Al finalizar la práctica, serás capaz de construir un mini-SDK modular en Python que permita cargar configuraciones desde un archivo JSON y procesar métricas de sensores de forma estructurada. Asimismo, implementarás un manejo profesional de errores mediante el uso de excepciones personalizadas, logging y estructuras try-except-else-finally, garantizando mayor robustez y trazabilidad en el código. Finalmente, podrás definir contratos claros de retorno en cada función y aplicar buenas prácticas de calidad propias del desarrollo de SDKs orientados a servicios en la nube, fortaleciendo la mantenibilidad y escalabilidad de tus soluciones..

  - ⏱️**Duración estimada**: 25 min.



### [Laboratorio 4. Mini-SDK Modular con Entorno Reproducible](Capítulo 4/README.md)

  - **Descripción**: Al finalizar la práctica, serás capaz de estructurar un proyecto Python con un enfoque profesional, organizándolo en módulos con responsabilidades claramente separadas. Además, podrás gestionar dependencias de manera adecuada mediante el uso de entornos virtuales (venv) y archivos requirements.txt, asegurando la reproducibilidad del entorno de desarrollo. Asimismo, aplicarás librerías como pathlib, os y sys para desarrollar código portable, culminando en la construcción de un mini-SDK funcional que integre un flujo completo de configuración, procesamiento de datos y simulación de envío a una API REST..

  - ⏱️**Duración estimada**: 25 min.



### [Laboratorio 5. Mini-SDK Profesional de Métricas en la Nube](Capítulo 5/README.md)

  - **Descripción**: Al finalizar la práctica, serás capaz de diseñar un mini-SDK en Python orientado a objetos, aplicando principios como jerarquías de clases, encapsulamiento, uso de propiedades y herencia para estructurar soluciones robustas. Asimismo, implementarás un manejo profesional de errores HTTP mediante excepciones personalizadas y construirás un cliente REST utilizando la librería requests. Finalmente, podrás analizar y probar llamadas HTTP empleando herramientas como unittest.mock.patch, lo que te permitirá validar el comportamiento del SDK sin necesidad de contar con un servidor en ejecución, fortaleciendo así las buenas prácticas de pruebas y desarrollo..

  - ⏱️**Duración estimada**: 30 min.
